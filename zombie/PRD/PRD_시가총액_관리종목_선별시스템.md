# PRD — 시가총액 관리종목 지정 우려기업 선별 시스템
**Market Cap Risk Screening System** | v1.0 | 2026년 3월

| 항목 | 내용 |
|------|------|
| 목적 | 상장폐지 개혁안 기준 관리종목 지정 우려기업 자동 선별 |
| 데이터 제공 | 사용자 제공 CSV (스냅샷 + 위험군 이력) |
| 코드 구현 | 별도 단계에서 진행 |

---

## 1. 배경

2026년 4월 1일부터 시행되는 상장폐지 개혁안(금융위원회)은 코스피·코스닥 시장의 부실기업 퇴출 기준을 대폭 강화한다. 특히 시가총액 요건은 2026년 7월 1일부터 새 기준이 적용되어 기존 대비 4~6배 높아진다. 이 시스템은 새 기준 시행 전에 위험군 기업을 선제적으로 파악하여 기사 작성 및 투자자 경고에 활용하기 위해 개발한다.

### 1.1 시가총액 요건 변경 내용

| 시장 | 현행 기준 | 2026.7.1 신기준 | 2027.1.1 강화 |
|------|----------|----------------|--------------|
| 코스닥 | 40억 원 미만 | 200억 원 미만 | 300억 원 미만 |
| 코스피 | 50억 원 미만 | 300억 원 미만 | 500억 원 미만 |

### 1.2 관리종목 지정 프로세스

- 관리종목 지정: 30거래일 연속 시총 기준치 미달 시 지정
- 상장폐지: 지정 후 90거래일 내 연속 45거래일 이상 회복 못하면 즉시 상장폐지

---

## 2. 목표

- 2026년 7월 1일 신규 기준 적용 시 관리종목 지정 위험에 처할 코스피·코스닥 기업을 자동으로 선별
- 위험 등급(🔴🟠🟡🟢)을 부여하고, 위험군 종목의 시총 추세를 분석
- ETF·ETN·스팩·리츠 등 요건 비적용 종목 자동 제외
- 결과를 CSV로 저장하여 기사 작성 및 후속 추적에 활용

---

## 3. 데이터 입력

### 3.1 1단계 입력: 스냅샷 데이터

사용자가 직전 영업일 기준 전종목 데이터를 CSV로 제공한다.

| 컬럼 | 타입 | 설명 |
|------|------|------|
| market | string | 시장 구분 (KOSPI / KOSDAQ) |
| ticker | string | 종목코드 6자리 |
| name | string | 종목명 (ETF·스팩 필터링에 사용) |
| base_date | string | 기준일자 (YYYYMMDD) |
| close_price | int | 기준일 종가 (원) |
| listed_shares | int | 상장주식수 |
| market_cap | int | 시가총액 (원) — close_price × listed_shares |

> 이 데이터만으로 Step 1(종목 구성), Step 2(위험 등급 분류) 완료 가능

### 3.2 2단계 입력: 위험군 이력 데이터

1단계에서 🔴🟠🟡로 분류된 종목에 한해, 해당 종목들의 45거래일치 일별 시총 이력을 사용자가 추가 제공한다. 전종목 이력을 처음부터 제공할 필요 없이 위험군만 추리므로 데이터 규모를 최소화한다.

| 컬럼 | 타입 | 설명 |
|------|------|------|
| market | string | 시장 구분 |
| ticker | string | 종목코드 6자리 |
| name | string | 종목명 |
| base_date | string | 해당 거래일 (YYYYMMDD) |
| close_price | int | 해당일 종가 |
| listed_shares | int | 상장주식수 |
| market_cap | int | 해당일 시가총액 |

> 45거래일을 선택한 이유: 관리종목 지정 트리거(30거래일) + 추세 확인용 여유분(15거래일)

## 4. 핵심 로직

### 4.1 Step 1 — 종목 Universe 구성

- 스냅샷 CSV 로드
- 제외 종목 필터링: 종목명에 ETF·ETN·스팩·SPAC·리츠·REIT 포함 시 제외
- 코스피·코스닥 구분 유지

### 4.2 Step 2 — 위험 등급 분류

2026년 7월 1일 신규 기준치를 적용하여 현재 시총 기준으로 4단계 등급을 부여한다. 현행 기준이 아닌 신기준을 사용하는 이유는, 지금 정상인 기업 중 7월 이후 위험군에 편입될 기업을 선제적으로 파악하는 것이 목적이기 때문이다.

| 등급 | 코스닥 기준 | 코스피 기준 | 의미 |
|------|-----------|-----------|------|
| 🔴 위험 | 200억 미만 | 300억 미만 | 7월 1일부터 즉시 30거래일 카운트 시작 |
| 🟠 경고 | 200억 ~ 260억 | 300억 ~ 390억 | 기준치 130% 미만 — 한 달 내 하락 시 미달 가능 |
| 🟡 주의 | 260억 ~ 400억 | 390억 ~ 600억 | 기준치 200% 미만 — 급락 시 경고권 진입 가능 |
| 🟢 안전 | 400억 초과 | 600억 초과 | 단기 우려 낮음 |

> 🟠 경고 기준을 130%로 설정한 이유: 코스닥 소형주는 한 달(20거래일) 안에 20~30% 하락이 현실적으로 가능한 범위이기 때문

### 4.3 Step 3 — 추세 분석 (위험군 종목만)

🔴🟠🟡 등급 종목에 한해 2단계 이력 데이터로 아래 항목을 계산한다.

| 항목 | 계산 방법 | 의미 |
|------|----------|------|
| 연속 미달일 | 이력을 역순 순회하며 신기준치 미달 연속일 카운트 | 30일 이상이면 관리종목 지정 임박 |
| 추세 방향 | 45거래일 전 대비 시총 변화율로 상승/보합/하락 분류 | 하락 추세 여부 확인 |
| 45일대비(%) | (현재 시총 - 45거래일 전 시총) / 45거래일 전 시총 × 100 | 구체적 변화폭 수치 |

---

## 5. 출력 결과

### 5.1 출력 컬럼

| 컬럼 | 내용 |
|------|------|
| ticker | 종목코드 |
| name | 종목명 |
| market | KOSPI / KOSDAQ |
| market_cap(억) | 현재 시가총액 (억 원, 소수점 1자리) |
| threshold(억) | 적용 기준치 (코스닥 200억 / 코스피 300억) |
| 위험등급 | 🔴 / 🟠 / 🟡 / 🟢 |
| 추세 | 상승 / 보합 / 하락 |
| 연속미달일 | 신기준치 연속 미달 거래일 수 |
| 45일대비(%) | 45거래일 전 대비 시총 변화율 |

### 5.2 정렬 및 저장

- 정렬: 위험등급 오름차순 → 시가총액 오름차순
- 저장 형식: CSV (UTF-8 BOM, Excel 호환)
- 파일명: `market_cap_risk_YYYYMMDD.csv`

---

## 6. 제약 및 가정

- 기준일자는 당일이 아닌 직전 영업일을 사용 (당일 데이터는 장 마감 오후 6시 이후 유효)
- 스팩 제외는 종목명 키워드 기반이므로 키워드에 포함되지 않는 스팩이 일부 누락될 수 있음
- 2027년 1월 1일 추가 강화 기준(코스닥 300억, 코스피 500억)은 이 버전에서 별도 처리하지 않음
- market_cap 컬럼을 직접 사용하므로 listed_shares_source 차이에 따른 영향 없음
- 현행 기준이 아닌 2026년 7월 1일 신기준 기준으로만 동작함
