# Agent Working Rules

## 1. Proactive Coding Restriction
- **Rule**: Do not use coding tools (file modification, command execution) without explicit approval.
- **Process**: Always prioritize planning and conceptual explanations first.
- **User Trigger**: "앞으로 코딩 도구는 내가 허락할 때만 사용하고, 먼저 계획이나 개념적 설명만 해줘"

## 2. Code Review and Approval
- **Rule**: Proposals for code changes must be shown in text first. Only apply changes to files after the user says "OK" or provides explicit permission.
- **Process**: Plan -> Suggest Code -> Wait for Approval -> Apply Changes.
- **User Trigger**: "수정할 내용을 코드로 먼저 보여주고, 내가 OK 하면 파일에 반영해"

---
*These rules apply to all AI agents interacting with this repository.*
